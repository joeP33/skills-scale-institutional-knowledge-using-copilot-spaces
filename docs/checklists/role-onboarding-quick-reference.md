# Role Onboarding Quick Reference

Use this quick reference for new hires and cross-team collaboration. It summarizes key responsibilities and primary contacts for each persona.

## Project Manager
- Key responsibilities:
  - Coordinate plans, dependencies, and risks
  - Run delivery cadence and project reporting
  - Own stakeholder communication flow
- Primary contacts: Product Manager, DevOps Engineer, QA Lead, Stakeholders

## Product Manager
- Key responsibilities:
  - Define outcomes and prioritize backlog
  - Own acceptance criteria quality and scope decisions
  - Align roadmap with stakeholders
- Primary contacts: Project Manager, Business Analyst, UX Designer, Developers

## Developers
- Key responsibilities:
  - Build, review, and test implementation changes
  - Raise technical risks and estimate work
  - Support release and post-deploy triage
- Primary contacts: Product Manager, QA Lead, DevOps Engineer

## QA Lead
- Key responsibilities:
  - Define test strategy and quality gates
  - Coordinate test execution and defect triage
  - Recommend release go/no-go from quality perspective
- Primary contacts: Product Manager, Developers, Project Manager, DevOps Engineer

## UX Designer
- Key responsibilities:
  - Lead discovery and usability validation
  - Deliver user flows, wireframes, and prototypes
  - Support design QA and accessibility decisions
- Primary contacts: Product Manager, Business Analyst, Developers, QA Lead

## Data Analyst
- Key responsibilities:
  - Define success metrics and measurement plans
  - Validate telemetry/data quality and reporting
  - Analyze outcomes and recommend optimizations
- Primary contacts: Product Manager, Developers, Project Manager

## DevOps Engineer
- Key responsibilities:
  - Maintain CI/CD and deployment runbooks
  - Ensure environment readiness and rollback capability
  - Monitor reliability and respond to incidents
- Primary contacts: Developers, QA Lead, Project Manager

## Business Analyst
- Key responsibilities:
  - Elicit and document business requirements
  - Map workflows, edge cases, and dependencies
  - Trace requirements to acceptance criteria and tests
- Primary contacts: Product Manager, Project Manager, UX Designer, QA Lead

## Stakeholders
- Key responsibilities:
  - Provide business context and priority inputs
  - Review milestones and decision points
  - Confirm outcomes and adoption considerations
- Primary contacts: Project Manager, Product Manager, Business Analyst
